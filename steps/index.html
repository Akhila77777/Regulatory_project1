From project to job offer

DermaScan is an AI skin-lesion classifier built as if it were going through EU conformity assessment as a Software as a Medical Device. Each section below is both a build phase of the project and a study topic for interviews. Work them in order: regulatory logic flows from intended purpose to risk to requirements to verification to post-market.

Why this project is well-timed (2026)
DermaScan sits exactly on the most contested skill in European MedTech right now: Regulatory Affairs + AI dual-fluency. It is the textbook case for the EU AI Act applied to an MDR device. Use that framing in every application.

The regulatory stack, at a glance

How to use this with the project

  • Build the repo in the order of the sections (see CLAUDE.md and the dermascan-ra skill).
  • Each page ends with interview Q&A - drill those before any RA interview.
  • Callout legend: REG the actual regulation, HOT 2026 current trend, TIP how to do it, WATCH OUT common mistake, QUIZ test yourself.
  • Pages are print-friendly - use the browser's print to make PDF revision sheets.
Honesty note
DermaScan is a training/simulation project, never a real device. Present it that way in interviews - showing you understand why the rules exist is more impressive than pretending you certified something.
